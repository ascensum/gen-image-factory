## Implementation Details

### Full-Text Search Implementation

**Plugin**: `@easyops-cn/docusaurus-search-local`

**Installation**:
```bash
cd website
npm install @easyops-cn/docusaurus-search-local
```

**Configuration** (`website/docusaurus.config.js`):
```javascript
plugins: [
  [
    require.resolve('@easyops-cn/docusaurus-search-local'),
    {
      indexDocs: true,
      indexBlog: false,
      indexPages: true,
      hashed: true,
      language: ['en'],
      highlightSearchTermsOnTargetPage: true,
    },
  ],
],
```

**Industrial Zen Styling** (`website/src/css/custom.css`):
- Search Input: Glassmorphism style matching navbar
- Search Popover: Dark card background (#0d0d0d) with Shiftline Blue accents
- Monospace typography for all search results
- Neutral gray borders matching bento cards
- Keyboard shortcuts automatically adapt to OS (⌘K on Mac, Ctrl+K on Windows/Linux)

## Implementation History

### Mobile Menu Fixes (2025-01-28)

**Initial Issue**: Mobile menu not working on pages other than homepage due to invalid props added by previous agent.

**Complete Resolution**:
- **Root Causes**:
  1. Backdrop-filter on parent `.navbar` breaking mobile sidebar (Docusaurus issue #6996)
  2. Contextual sidebar on docs pages hiding primary navigation
  3. Component mismatch (Content vs PrimaryMenu)
- **Fixes Applied**:
  - Created `PrimaryMenu` component to force global navbar items
  - Created `Layout` component to prevent contextual sidebar
  - Moved backdrop-filter to desktop-only media query (>= 996px)
  - Applied Industrial Zen styling (monospace, uppercase, letter-spacing)
- **Result**: Mobile menu fully functional on all pages with consistent styling
