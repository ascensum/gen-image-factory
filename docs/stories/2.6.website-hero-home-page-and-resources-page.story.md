### Hero Home Page React Component

**Template Source**: `.superdesign/design_iterations/hero_page_template.html` and `.superdesign/design_iterations/hero_theme.css`

**File Location**: `website/src/pages/index.tsx`

**Key Conversion Steps**:

1. **Create the React Component**:
   - Create `website/src/pages/index.tsx` (this will override the default docs index page)
   - Use TypeScript for type safety
   - Import necessary Docusaurus components and hooks

2. **Convert HTML to JSX**:
   - Change `class` to `className`
   - Change `for` to `htmlFor`
   - Self-close tags (`<br />` instead of `<br>`)
   - Convert inline event handlers to React event handlers
   - Use JSX expressions for dynamic content

3. **Handle CSS - Site-Wide Styling**:
   - Update `website/src/css/custom.css` to implement permanent dark mode styling
   - All pages use consistent dark theme (#050505 background)
   - See "Visual Overhaul - Permanent Dark Mode" section for styling details

4. **Docusaurus-Specific Changes**:
   - **Theme is locked to dark mode** - No theme toggle available
   - **Use Docusaurus Link component** for internal navigation:
     ```tsx
     import Link from '@docusaurus/Link';
     // Use: <Link to="/getting-started/installation">Documentation</Link>
     ```
   - **Use regular `<a>` tags** for external links (GitHub, email, etc.)

5. **Icons**:
   - **Option 1**: Install Lucide React: `npm install lucide-react` in `website/` directory
     ```tsx
     import { Download, BookOpen, Store, BrainCircuit, ShieldCheck, Zap, Database, Monitor } from 'lucide-react';
     ```
   - **Option 2**: Use SVG icons directly or Docusaurus icon components
   - **Option 3**: Use simple emoji or Unicode symbols as fallback

6. **Component Structure**:
   ```tsx
   import React from 'react';
   import Layout from '@theme/Layout';
   import Link from '@docusaurus/Link';
   import useDocusaurusContext from '@docusaurus/useDocusaurusContext';
   import './hero-theme.css'; // or import in custom.css
   
   export default function Home(): JSX.Element {
     const {siteConfig} = useDocusaurusContext();
     
     return (
       <Layout title="Home" description={siteConfig.tagline}>
         <main>
           {/* Hero Section */}
           {/* Features Section */}
           {/* Product Showcase */}
           {/* Footer */}
         </main>
       </Layout>
     );
   }
   ```

7. **Key Sections to Convert**:
   - **Hero Section**: Logo placeholder, title, subtitle, CTA buttons, version badge, Blue Lamp Glow, SVG grid
   - **Features Grid**: Asymmetrical Bento Grid layout with 6 feature cards (Industrial Glass styling)
   - **Product Showcase**: Screenshot placeholder section
   - **Note**: Custom footer removed - using Docusaurus footer only

8. **Animations**:
   - Keep CSS animations (`fade-in-up`, `pulse-subtle`)
   - Use `IntersectionObserver` for scroll-triggered animations (already in template)
   - Consider using `framer-motion` for more advanced animations if needed

9. **Responsive Design**:
   - The template already includes responsive CSS
   - Test on mobile (320px+), tablet (768px+), desktop (1024px+)
   - Ensure buttons stack on mobile, horizontal on desktop

10. **Accessibility**:
    - Keep all ARIA labels from template
    - Ensure keyboard navigation works
    - Maintain proper focus states
    - Test with screen readers

11. **Links to Update**:
    - Download: `#download` (placeholder - update when download page exists)
    - Documentation: `/getting-started/installation` (use Docusaurus Link)
    - Microsoft Store: `#microsoft-store` (placeholder - update when available)
    - GitHub: `https://github.com/ShiftlineTools/gen-image-factory` (external link)
    - Support Email: `mailto:admin@shiftlinetools.com` (external link)

12. **Docusaurus Routing**:
    - Creating `website/src/pages/index.tsx` automatically makes it the home page
    - No additional routing configuration needed
    - The page will be available at `/` (root)

13. **Version Badge**:
    - Monospaced pill badge displaying app version: `[ 1.0.8 ]`
    - Positioned directly under the "Gen Image Factory" heading
    - Automatically fetches latest version from GitHub Releases API
    - Version fetching script (`website/scripts/fetch-version.js`) runs automatically before build/start
    - Falls back to `package.json` version if GitHub API is unavailable
    - Version is cached in `.version-cache.json` (1-hour TTL) to reduce API calls
    - Version badge styled with gray background and border matching the design system
