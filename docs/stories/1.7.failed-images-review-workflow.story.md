# Story 1.7: Failed Images Review Workflow

## Status: Done

## Story

**As a** Content Creator/Quality Manager,  
**I want** to review images that failed quality assurance checks,  
**so that** I can manually approve usable images that AI QA incorrectly flagged and manage images that failed post-processing steps.

## Acceptance Criteria

1. **‚úÖ Create failed images review interface** - Display all images with QC status 'failed' in a dedicated review page linked from main dashboard
2. **‚úÖ Display QC failure reasons** - Show QC failure reasons prominently for each failed image (copyright/trademark issues, text misspellings, etc.)
3. **‚úÖ Implement manual approval workflow** - Allow users to manually approve failed images (change status from 'failed' to 'approved') - **approved images automatically appear in main Dashboard Panel as 'Success' images**
4. **‚úÖ Add simple retry mechanism for post-processing failures** - Enable retry of failed post-processing (background removal, metadata creation) with **either original settings OR modified post-processing settings for entire batch** (not both, no individual image settings)
5. **‚úÖ Style review components with Tailwind CSS** - Ensure consistent styling with the rest of the application
